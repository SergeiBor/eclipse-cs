Using custom checks

One great feature of Checkstyle is its extentablity. You can easily write own checks as explained here.

Once you have written your check you surely want to use them with the plugin.

There are a few steps you must take to accomplish this:

  1. Package your checks into a .jar file

  2. Go to <path-to-my-eclipse>/plugins/com.atlassw.tools.eclipse.checkstyle_x.x.x directory and copy the .jar into the extension-libraries directory

  3. Open the META-INF/MANIFEST.MF file from within the with an text editor

  4. Extend the Bundle-ClassPath entry to include your .jar file
    Before: Bundle-ClassPath: CheckstylePlugin.jar,NLS.jar,checkstyle-all-4.0-beta2.jar,extension-libraries/checkstyle-optional-4.0-beta2.jar
    After: Bundle-ClassPath: CheckstylePlugin.jar,NLS.jar,checkstyle-all-4.0-beta2.jar,extension-libraries/checkstyle-optional-4.0-beta2.jar,extension-libraries/myCustomChecks.jar

  5. Restart Eclipse with the -clean option – this is needed to let Eclipse pick up the changes you made

  6. Write a Checkstyle configuration file using a text editor – include your custom modules within this configuration.
    Be sure to include all properties of your modules within the module configuration so you can edit the properties afterward in the Checkstyle Plug-in Configuration Editor.
    Note that the custom check will show up in the Other section of the configuration editor.
    You are now able to configure the known properties of your check and to remove the check from the configuration.
    After you removed it you can't add it again using the Checkstyle Plug-in Configuration Editor – so be careful.

    This limitation exists because there is no meta data for you check known to the plug-in.
    If you are really knowing what you are doing you might want to try fiddling with the module metadata within the CheckstylePlugin.jar.

    Maybe a later release will let you specifiy the meta data for your checks more easily.
